Class SelLog

Logging facility.

If connected to a broker, in addition to the destination set in init() function, the message is published as well on "clientID/Log/level" topic.

Following level are automatically registered :

  • 'F' : "Fatal"
  • 'E' : "Error"
  • 'W' : "Warning"
  • 'T' : "Trace"

Methods

SelLog:init (filename, where)

Initialise logging

Notez-bien :

  • Whatever the value of where, the message is always published if connected to a broker.
SelLog:register (level, topic) Register a logging level
SelLog:ignore (levels) Ignore logging levels
SelLog:log (level, message) Log a message
SelLog:status () Return the current status of logging


Methods

SelLog:init (filename, where)

Initialise logging

Notez-bien :

  • Whatever the value of where, the message is always published if connected to a broker.
  • depending on logging level, the message is output on stdout or stderr

Parameters:

  • filename string file to log to
  • where boolean true both on stdout and file, false only on stdout, unset only on file
SelLog:register (level, topic)
Register a logging level

Parameters:

  • level string (only the 1st char is took in account)
  • topic string corresponding topic extension
SelLog:ignore (levels)
Ignore logging levels

Parameters:

  • levels string list of all levels to ignore

Usage:

    -- to ignore Trace and Warning
 SelLog.ignore("TW")
SelLog:log (level, message)
Log a message

Parameters:

  • level string log level as registered (optional, 'I' by default)
  • message string text to log

Usage:

    SelLog.log("my informative message")
SelLog.log('E', "my error message")
SelLog:status ()
Return the current status of logging

Returns:

  1. boolean file logging on file
  2. boolean std logging on STDOUT/STDERR
